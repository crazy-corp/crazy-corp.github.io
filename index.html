OPTIMISE IMAGES

The concept is simple: We decide a max resolution required for our image on the website and we compress the image if the resolution of the image is higher than the max resolution variable. The Image.Resampling.LANCZOS() method is used so that image doesn’t lose its aspect ratio while compressing.

Will this cause a decrease in quality of images?

That’s the interesting part, Do we need a pic with 7000*5000 px resolution on a website in which the desktop size is 1280 x 1024 (Super-eXtended Graphics Array (SXGA)). We can still store the original image if necessary. But there are millions of websites in which the images are used in an improper manner. Most common in the gallery page of websites. I picked a max width of 1600px which is huge. Still I achieved this optimized result. Websites like instagram and twitter use such image optimisation techniques. My algorithm can be used by any websites in which django is used as a backend. Without any complexity, we can reduce the carbon emission of millions of websites.

BUS TRACKING

The project proposes a model where optimum utilization of existing resources takes place and at the same time reduces internet traffic as well as save costs for the operator. It consists of three main important components, Node, Gateway, Web Application. The bus stop node has STM32 microcontroller connected to the Lora and PN532 RFID Reader.The buses going through the bus stops have RFID Tags on it ,eventually the RFID reader reads the RFID tag data(bus id) which is unique for each bus. The received data is transmitted to the nearby Gateway and transmission confirmation is done on reception of acknowledgement from the gateway.
